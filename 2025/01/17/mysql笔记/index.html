MySQL语句基本运行顺序

MySQL的SQL语句执行顺序大致如下：

  1. FROM子句：确定数据来源，包括JOIN的表。
  2. ON：执行JOIN条件。
  3. JOIN：如果有JOIN则进行JOIN操作。
  4. WHERE子句：执行过滤条件。
  5. GROUP BY子句：根据指定的列分组结果集合。
  6. HAVING子句：执行分组过滤条件。
  7. SELECT子句：选取特定的列。
  8. DISTINCT子句：去除重复数据。
  9. ORDER BY子句：最后对结果进行排序。
  10. LIMIT子句：限制结果集数量。

MySQL 储存数据类型

char 和 varchar 的区别：

  • ‌存储长度‌：char是一种固定长度的数据类型，需要指定字段的长度。当存储数据时，char会将数据填充到指定的长度，如果数据长度小于指定长度，则使用空格进行填充。例如，char(10)表示该字段长度为10个字符，无论实际数据长度如何，都会占用10个字符的空间。相比之下，varchar是一种可变长度的数据类型，也需要指定字段的最大长度，但实际存储的数据只会占用所需的空间。例如，varchar(10)表示该字段最多可以存储10个字符，实际存储空间根据数据的长度变化。‌
  • ‌性能‌：由于char的固定长度特性，其在存储和检索过程中的性能较好，存取速度较快，但可能会浪费存储空间。char类型的字段在存取速度上甚至可以比varchar快50%。相反，varchar虽然存取速度较慢，但由于其长度不固定，可以节省存储空间。‌
  • ‌使用场景‌：由于char类型的性能较好且固定长度，适合存储长度固定的数据，如身份证号码、国家代码等。而varchar适合存储长度可变的数据，如姓名、地址等。

CASE 语句

CASE 语句的结构：

CASE 语句用于在查询中进行条件判断，根据不同的条件返回不同的值。END 是 CASE 语句的结束标志，而 AS 用来给返回的结果列命名。

END AS 的作用：

  • END: 用于标识 CASE 语句的结束，告诉 SQL 引擎该语句已经完成并准备返回一个结果。
  • AS: 用来给 CASE 语句的返回结果列起一个别名，这样在结果集中可以以更易读的方式引用该列。
